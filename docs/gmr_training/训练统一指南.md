# GVHMR→ GMR → IsaacLab 训练统一指南

## 1) GVHMR模块

### 环境检查
```bash
conda activate env_isaaclab
```

### WandB 配置
```bash
wandb login
```
网页端创建 Registry Collection：
- Core → Registry → New Collection
- Name: Motions
- Type: All Types

Registry 路径格式：
`<username>-org/wandb-registry-motions/<motion_name>:<version>`

## 5) 参数速查

### csv_to_npz.py
- `--input_file`（必需）
- `--output_name`（必需）
- `--input_fps`（默认 30）
- `--output_fps`（默认 50）
- `--frame_range START END`（可选）
- `--headless`

### train.py
- `--registry_name`（必需）
- `--task`
- `--num_envs`
- `--max_iterations`
- `--headless`
- `--logger`
- `--log_project_name`

---

## 6) 推荐配置

| 动作类型 | num_envs | max_iterations | 预计时间 |
|---|---:|---:|---:|
| 简单（stand） | 4096 | 500 | ~1h |
| 中等（walk/turn） | 4096 | 1000 | ~2h |
| 复杂（crouch） | 4096 | 1500 | ~3h |
| 高难（dance） | 8192 | 2000 | ~5h |

显存不足优先降低 `--num_envs`（如 4096 → 2048）。

## 7) 常见问题

### CSV 不存在
检查 `/home/abc/GMR/pkl_files/csv/`，为空则先在 GMR 侧生成。

### WandB 上传失败
```bash
ping api.wandb.ai
wandb login --relogin
python -c "import wandb; wandb.Api(); print('OK')"
```

### Registry 路径错误
正确：`username-org/wandb-registry-motions/motion_name:latest`

### train.py 找不到动作
- 确认 `csv_to_npz.py` 已成功执行并上传
- 确认 `--registry_name` 与上传的 `output_name` 一致

### GPU 显存不足
降低 `--num_envs`，必要时再降低并行规模。
